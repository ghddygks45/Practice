add -> commit -> push

pull : 내려받기

commit 내용,
1. 첫줄에 간단하지만 명확한 내용을 쓴다.
2. 한 줄 비우고
3. 자세한 내용을 적는다.

파일 내용 되돌리기
- 특정 파일의 내용을 마지막 커밋으로 돌리고 싶을 때 '코드뭉치 버리기' 선택

특정커밋으로 돌아가고 싶을 때, 해당커밋에서 브랜치를 만든다.
그 이후 만든 브랜치에서 다른작업을 할 수 있다.
checkout : 특정 브랜치로 돌아간다. '소스트리에선 더블클릭할 수 있다'


merge : 병합

병합할 때는 마스터 브랜치로 돌아가서 version2 브랜치를 병합한다.(마우스 우클릭)


conflict : 충돌

만약 version2와 version3를 만들어 브랜치를 하나씩 병합할 경우,
version2의version3의 같은 내용을 다르게 수정했다면 충돌이 일어난다.

충돌해결법,

1. 에디트에서 수동으로 충돌나는부분을 지워준다.
2. 커밋하지않은 변경사항 클릭, 스테이지에 올라간내용(마우스 우클릭) -> 충돌해결 -> 내것 or 저장소것 클릭.
versiwon3에서 임시파일을 하나 만들어서 수정한다. 그리고 마스터에서 version2와 version3를 순차적으로 병합한다.
그럼 version3의 파일이 아예다른파일이기 때문에 충돌이 일어나지 않는다.

병합을 한 다음엔 브랜치를 삭제해도 무방하다.


커밋되돌리기

1. 브랜치를 만들어서 커밋 되돌리기 : 단순히 브랜치를 하나 만들고 내용을 만든다. 후에 병합함.
2. revert 마우스오른쪽 커밋되돌리기 : 이전 커밋을 남기고 되돌린다.(내용은 같지만 커밋이 보존된다.)
3. revert로 여러커밋 되돌리기 : 최신순서부터 차례로 커밋을 되돌리면 된다.
